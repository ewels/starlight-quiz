`starlight-quiz` ships a small CLI for working with quizzes outside the browser. Both commands read the
[quiz manifest](/starlight-quiz/guides/configuration/#manifest) (never your MDX), so enable it first:

```js title="astro.config.mjs"
starlight({ plugins: [starlightQuiz({ manifest: true })] });
```

The manifest (`quiz-manifest.json`) is written to your build output, and is served at the root of your deployed site.

## Take a quiz in the terminal

Run quizzes interactively from a local build or a deployed site:

```sh
# From a local build directory (or a manifest file directly)
npx starlight-quiz run ./dist

# From a deployed site, its manifest is fetched automatically
npx starlight-quiz run https://my-docs.example.com
```

The runner presents each question, grades your answers, reveals the correct ones when you are wrong, and prints a final
score with a tier message. If a site has no manifest, the CLI falls back to scraping the page's HTML.

Add `--shuffle` to randomise the quiz order and the answer order within each quiz, or `--shuffle-answers` to randomise
only the answers. Use `--page <substr>` to take just the quizzes whose page path contains a string:

```sh
npx starlight-quiz run ./dist --shuffle
npx starlight-quiz run ./dist --page /guides/multiple-choice/
```

## History

Every run is recorded to `~/.local/share/starlight-quiz/history.json` (honouring `XDG_DATA_HOME`). Review or clear it:

```sh
npx starlight-quiz history          # a table of past runs
npx starlight-quiz history --json   # the raw JSON
npx starlight-quiz history --yaml   # the same as YAML
npx starlight-quiz history --clear  # wipe it
```

## Export to QTI for an LMS

Export every quiz as [QTI](https://www.imsglobal.org/question/index.html) assessment items, the interchange format
understood by Canvas, Moodle, Blackboard and others:

```sh
# QTI 2.1 (default)
npx starlight-quiz export-qti ./dist --out ./qti

# QTI 1.2
npx starlight-quiz export-qti ./dist --version 1.2 --out ./qti
```

This writes one XML item per quiz plus an `imsmanifest.xml` into the output directory, an unzipped IMS Content
Package. Zip the folder to import it into your LMS. Single-choice, multiple-choice and fill-in-the-blank questions are
all supported.

## Options

| Command           | Argument / flag      | Description                                               |
| ----------------- | -------------------- | --------------------------------------------------------- |
| `run`             | `<source>`            | A manifest file, a directory containing one, or a site URL. |
| `run`             | `--shuffle`           | Randomise quiz and answer order.                          |
| `run`             | `--shuffle-answers`   | Randomise only the answer order within each quiz.         |
| `run`             | `--page <substr>`     | Only run quizzes whose page path contains `<substr>`.     |
| `run`             | `--filename`          | Manifest filename (default `quiz-manifest.json`).         |
| `export-qti`      | `<source>`            | As above.                                                 |
| `export-qti`      | `--out`               | Output directory (default `qti`).                         |
| `export-qti`      | `--version`           | `2.1` (default) or `1.2`.                                 |
| `history`         | `--json` · `--yaml` · `--clear` | Print past runs as JSON · as YAML · delete the saved history. |
