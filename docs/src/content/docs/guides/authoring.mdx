title: Authoring quizzes
description: The full markdown syntax for writing quizzes.

import { Quiz } from 'starlight-quiz/components';

Everything inside a `<Quiz>` is ordinary markdown. There is no new syntax to learn beyond a few conventions.

## Answers

Write answers as a GFM task list. A ticked box (`[x]`) is a correct answer; an empty box (`[ ]`) is incorrect.

```mdx
<Quiz title="Single choice">
What is the capital of France?

- [ ] London
- [x] Paris
- [ ] Berlin
</Quiz>
```

<Quiz title="Single choice" id="authoring-single">
What is the capital of France?

- [ ] London
- [x] Paris
- [ ] Berlin

</Quiz>

When **more than one** answer is correct, the quiz automatically becomes multiple-choice with checkboxes.

<Quiz title="Multiple choice" id="authoring-multiple">
Which of these are programming languages?

- [x] Rust
- [x] Python
- [ ] HTML
  > HTML is a markup language, not a programming language.
- [x] TypeScript

</Quiz>

## Per-answer feedback

Add a blockquote (`>`) indented underneath an answer to show feedback for that specific choice after submitting.

```mdx
- [ ] HTML
  > HTML is a markup language, not a programming language.
```

## Fill-in-the-blank

Wrap the expected answer in double square brackets. Answers are case-insensitive and trimmed of surrounding
whitespace. You can have several blanks; all must be correct.

```mdx
<Quiz title="Fill in the blank">
The powerhouse of the cell is the [[mitochondria]].
</Quiz>
```

<Quiz title="Fill in the blank" id="authoring-blank">
The powerhouse of the cell is the [[mitochondria]].

Water is made of hydrogen and [[oxygen]].

</Quiz>

## Post-answer explanation

A horizontal rule (`---`) separates the question from an explanation that is revealed once the reader submits.

```mdx
<Quiz title="With explanation">
Is the Earth flat?

- [ ] Yes
- [x] No

---

The Earth is an oblate spheroid.
</Quiz>
```

<Quiz title="With explanation" id="authoring-explanation">
Is the Earth flat?

- [ ] Yes
- [x] No

---

The Earth is an oblate spheroid — slightly flattened at the poles.

</Quiz>

## Options

Pass props to `<Quiz>` to change its behaviour:

<Quiz title="Auto-submit and shuffle" id="authoring-options" autoSubmit shuffle disableAfterSubmit={false}>
Pick the odd one out.

- [ ] Apple
- [ ] Banana
- [x] Carrot
- [ ] Mango

</Quiz>

| Prop                 | Default | Description                                                  |
| -------------------- | ------- | ------------------------------------------------------------ |
| `id`                 | —       | Stable id for persistence (falls back to a hash of `title`). |
| `shuffle`            | `false` | Shuffle answer order on load.                                |
| `autoSubmit`         | `false` | Submit a single-choice quiz when an answer is chosen.        |
| `disableAfterSubmit` | `true`  | Lock after submitting; when `false`, shows a reset button.   |
| `showCorrect`        | `true`  | Reveal the correct answer(s) after a wrong submission.       |
